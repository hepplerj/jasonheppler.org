A *colophon* is a publishing tradition: the note at the back of a book explaining how it was made, who set the type, what paper was used. This is that note for the web.

## Built With

The site is generated by [Hugo](https://gohugo.io), a static site generator written in Go. Hugo compiles quickly, which is one of the reasons I've stayed with it since 2018. There is no database, no server-side rendering, no runtime dependencies. Every page is a pre-built HTML file.

Styles are written with [Tailwind CSS v4](https://tailwindcss.com) and processed through [PostCSS](https://postcss.org). The source code lives on [GitHub](https://github.com/hepplerj/jasonheppler.org) and deploys automatically to [Cloudflare](https://cloudflare.com) on every push to `main`.

## Typography

All typefaces are self-hosted.

**[Fraunces](https://fraunces.undercase.xyz)** by Undercase Type sets the body text. It's a "wonky" old-style soft serif with roots in early twentieth-century display type, expressive at large sizes and comfortable at long reading lengths.

**[Inter](https://rsms.me/inter/)** by Rasmus Andersson handles interface elements: navigation, labels, metadata, and monogram.

**[JetBrains Mono](https://www.jetbrains.com/lp/mono/)** by JetBrains sets code blocks and monospaced text.

## Writing Environment

Content lives as flat Markdown files in the site's repository. I write in [Obsidian](https://obsidian.md) and publish directly to the site. There is no CMS, no admin interface, no database---just text files in a folder, versioned with Git. This keeps the writing environment and the publishing environment close together, which matters to me.

## Design Decisions

The navigation lives in a slide-out panel triggered from the top right corner, keeping the reading surface uncluttered. A monogram in the top left links home.

The site supports **dark and light modes**, toggled with an Apple-style switch in the nav panel. The preference is stored in `localStorage` and applied immediately on page load, so there's no flash of the wrong theme. Font size can be adjusted between three sizes.

Performance and environmental footprint informed many choices: all fonts are self-hosted variable files, images use lazy loading and `decoding="async"`, scripts are deferred, and the page weight on a typical post is kept small. The goal is a page that loads quickly on a slow connection and generates as little CO₂ as possible.

## Content Types

**Essays** are longer, more considered pieces: things that took weeks or months to write and went through multiple drafts. They live at `/:year/:month/:day/:slug/`.

**Notes** are shorter observations, technical notes, and things I wanted to record but didn't need to expand into full essays. They share the same URL structure, which means a note could someday become an essay without breaking any links.

**Commonplace** posts are links to things I've read and wanted to mark. Named after the commonplace book tradition of collecting passages and references worth keeping.

**Books** are my reading log: what I've read, brief impressions, and metadata for looking things up later.

This is version 9 of the site, dating back to a WordPress site in 2008.
